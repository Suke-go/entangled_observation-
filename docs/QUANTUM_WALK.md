# ±Quantum: 量子システム設計書

DESIGN.md（概念・美学）、SPEC.md（技術仕様）の補完文書。
プロトタイピングで確定した **3層ハイブリッドアーキテクチャ** の
設計判断、数学的基盤、実装方針をまとめる。

---

## 1. 設計判断の経緯

### 1.1 問題: ハーフトーンの限界

DESIGN.md §4.3 は画像のハーフトーン表現（+密度=暗、−密度=明）を記述していた。
プロトタイピングの結果、以下の問題が判明した:

| 問題 | 原因 |
|------|------|
| 自然写真（森林）が認識困難 | 拡散テクスチャはエッジがなく、離散化に耐えない |
| +/− の「意味」が不明確 | 写真再現が目的化し、+/−が単なるピクセル代替に |
| 「観測で浮かび上がる」体験とのずれ | 画像は最初から存在し、観測前後の差が不十分 |

### 1.2 決定: 3層ハイブリッドアーキテクチャ

量子ウォーク（波のビジュアル）＋ 量子回路（ゲート操作・エンタングルメント）＋
実機量子コンピュータ（オプション）を組み合わせる。

```
Layer 1 (L1): ビジュアルエンジン — quantum-walk.js
  離散シュレディンガー方程式で波紋パターンを生成
  |ψ|² → +/− の密度分布

Layer 2 (L2): 量子回路エンジン — quantum-circuit (npm)
  SPEC.md §4 のタイル型量子シミュレーション
  Hadamard, CNOT, 射影測定, POVM
  A-B エンタングルメント

Layer 3 (L3): 量子実機（オプション） — IBM Quantum / Amazon Braket
  本物の量子ノイズを含む結果
  バッチ実行 → キャッシュ方式
```

## 2. 量子回路モデル（Layer 2）

### 2.1 SPEC.md §4 のタイル型量子シミュレーション

```javascript
// 各タイルが独立な k 量子ビット系
QuantumTile {
  circuit: QuantumCircuit(k)     // k=2〜4 量子ビット
  stateVector: Complex[2^k]      // 状態ベクトル
  entangledPartner: QuantumTile  // 対面ディスプレイの対応タイル
  measuredQubits: Map<int, '+' | '-'>
}
```

グリッドを 32×16 = 512 タイルに分割。各タイル内は k=2〜4 量子ビット。

**計算量**: 512 × 2^4 = 8,192 複素数 = 128 KB。ブラウザで十分。

### 2.2 ゲート操作

```
Hadamard(tile, qubit)           → 重ね合わせ生成 |0⟩ → |+⟩
CNOT(tileA, tileB, ctrl, tgt)  → エンタングルメント生成
partialMeasure(tile, qubit, s)  → 注視による弱測定 (POVM)
```

### 2.3 エンタングルメント

A-B 間の対応タイル同士を CNOT で結合:

```
|ψ_A⟩|ψ_B⟩ → CNOT_{A→B} → エンタングル状態
```

A側の測定が B側の状態を即座に更新（射影測定の非局所効果）。

## 3. 量子ウォーク（Layer 1）

### 3.1 格子上の離散シュレディンガー方程式

```
H_wave = -J Σ_{<i,j>} (|i⟩⟨j| + |j⟩⟨i|) + Σ_i V_i(t)|i⟩⟨i|
```

- ホッピング項（運動エネルギー）→ 波面の伝播
- 波源ポテンシャル V_i(t) → 同心波の生成
- RK2 積分、周期的境界条件、微小ダンピング

### 3.2 L1 ↔ L2 の接続

```
L2 (量子回路) の各タイルの状態 ρ:
  p_plus = ⟨+|ρ|+⟩         → タイル領域の +/− 密度
  coherence = |⟨+|ρ|−⟩|     → タイル領域の揺らぎ強度

  ↓ L1 (量子ウォーク) のパラメータに反映:
  - p_plus が高い → 波の振幅が大（明るい）
  - coherence が高い → 波の周波数が速い（活発な揺らぎ）
  - 崩壊済みタイル → 波動関数を凍結（鮮明な +/−）
```

## 4. 体験設計

| 層 | 体験 | 所要時間 | 数学的対応 |
|----|------|---------|-----------|
| 0 | 「揺らぐ何かが見える」 | 0秒 | L1: |ψ|² + L2: pPlusフリッカー |
| 1 | 「見ると止まる。像が出る」 | 5〜10秒 | L2: 射影測定 → 崩壊 → 離散確定 |
| 2 | 「見ていない場所はまた揺れる」 | 30秒〜 | L2: 再量子化 |
| 3 | 「隣の人が見た場所も止まる」 | 1分〜 | L2: CNOT エンタングルメント |

## 5. 現在の実装状態

### 5.1 ファイル構成

```
entangled_observation/
├── docs/
│   ├── DESIGN.md              # 概念・美学
│   ├── SPEC.md                # 技術仕様
│   └── QUANTUM_WALK.md        # 本文書
├── js/
│   ├── quantum-walk.js        # ✅ L1: 離散シュレディンガーソルバー
│   ├── static-preview.js      # ✅ L1+L2: 離散フリッカー + 画像ハーフトーン レンダラー
│   ├── quantum-tile.js        # ✅ L2: タイル型量子回路 + 画像輝度バイアス
│   ├── entanglement-client.js # ✅ A-B WebSocket 崩壊リレー
│   ├── image-loader.js        # ✅ 画像→タイル解像度輝度マップ
│   ├── halftone.js            # 🔄 (旧) 輝度→確率振幅
│   ├── measurement.js         # 🔄 (旧) マウスベース擬似測定
│   ├── grid.js                # 🔄 (旧) 汎用 +/− グリッド描画
│   └── main.js                # 🔄 (旧) メインデモ
├── images/
│   └── sea_001.png            # ✅ ソース画像（海面）
├── server.js                # ✅ WebSocket リレー + 静的ファイル配信
├── static.html                # ✅ QW プレビューページ
├── index.html                 # メインデモページ
└── style.css                  # 共通スタイル
```

### 5.2 実装済み

- L1: `quantum-walk.js` — 2D Schrödinger ソルバー (RK2, 周期BC, ダンピング)
- L2: `quantum-tile.js` — タイル型量子回路 (H, CNOT, POVM, 再量子化)
- L1↔L2 接続: `static-preview.js` — 離散フリッカー + 画像ハーフトーン
- WebSocket サーバー: `server.js` — A-B 崩壊リレー
- 画像ローダー: `image-loader.js` — 輝度マップ生成

### 5.3 未実装

- アイトラッキング (MediaPipe Face Mesh + Iris)
- L3: IBM Quantum / Amazon Braket
- WebGL レンダラー（パフォーマンス必要時）
- 展示レベルのチューニング
